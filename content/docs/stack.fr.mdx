# Ma stack

<p className="text-muted-foreground">Décembre 2025 - Armand Thuillart</p>

Voici les domaines où je suis le plus productif :

## 1. Frontend (TanStack et React)

J'ai travaillé avec [SvelteKit](https://kit.svelte.dev/), [React Router](https://reactrouter.com/), et [Next.js](https://nextjs.org/) au fil des années. Dernièrement, j'utilise [TanStack Start](https://tanstack.com/start/) et je ne reviendrai pas en arrière.

## 2. Styling (Tailwind CSS et shadcn/ui)

Créer des composants flexibles et accessibles est difficile. Soit vous utilisez (et étendez) une bibliothèque de composants, soit vous construisez la vôtre. C'est pourquoi j'utilise [shadcn/ui](https://ui.shadcn.com/).

Cette bibliothèque fournit des composants bien conçus et extensibles, construits sur des [primitives accessibles et non stylées](https://base-ui.com/). On y trouve les éléments de base comme les boutons et les champs de saisie, mais aussi les icônes, les graphiques, et même les thèmes personnalisés.

Les composants sont stylés avec [Tailwind CSS](https://tailwindcss.com/), la bibliothèque CSS qui fonctionne le mieux avec l'IA. Pourquoi ? Cela facilite la génération et l'édition de code avec les outils d'IA.

Tailwind utilise un compilateur pour générer uniquement les classes utilisées. Ainsi, bien que le framework CSS utilitaire contienne de nombreux noms de classes possibles, le compilateur n'inclut que les classes utilisées dans le fichier CSS compilé unique.

Si vous n'écrivez que du code Tailwind, votre bundle ne dépassera jamais l'ensemble total des classes Tailwind utilisées. Il est très peu probable que vous les utilisiez toutes. Cela signifie qu'il existe une limite supérieure fixe sur la taille du fichier CSS généré, qui est ensuite minifié, compressé et mis en cache pour les meilleures performances.

## 3. Backend (Convex)

J'utilise Convex comme backend. Convex facilite le travail avec les bases de données, de manière sûre au niveau des types et agréable. Je peux visualiser et modifier mes données depuis leur tableau de bord et intégrer des fonctionnalités complexes comme les crons, workflow et workpool avec juste quelques lignes de code en utilisant leurs [composants](https://convex.dev/components).

Ce que j'apprécie le plus ? Convex fonctionne parfaitement avec TypeScript.

## 4. IA (Warp et Cursor)

[Cursor](https://cursor.com/home) est mon éditeur de code de prédilection. Il m'aide à éditer, refactoriser et déboguer le code.

Il est utile pour les refactorisations fastidieuses, ou la restructuration et le reformatage de code. Grâce à sa connaissance à jour de TanStack, React, Convex et d'autres outils web, il me propose d'excellentes suggestions de code.

[Warp](https://warp.dev) est mon terminal de prédilection. Il m'assiste pour le contrôle de version, les pull requests et le déploiement en production.

Les outils d'IA ne sont pas encore parfaits, mais ils permettent de gagner beaucoup de temps.
